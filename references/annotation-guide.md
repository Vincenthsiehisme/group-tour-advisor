# annotation-guide.md — 知識標注說明

> 載入時機：PM 詢問標注意義、或需要決定如何處理某條帶標注的規則時讀取。

---

## 三種標注的定義

| 標注 | 意義 | 輸出風險時的處理方式 |
|------|------|-------------------|
| `[雄獅規則－待確認]` | 描述雄獅「應該」的做法，但尚未實際確認 | 風險描述加註「規則尚未確認，需先與業務/OP 釐清再設計」 |
| `[業界通例]` | 一般旅遊業慣例，不特定於雄獅 | 可作為提問方向，但不直接斷言為雄獅規則 |
| `[需工程確認]` | 技術或系統介面細節 | 風險描述加註「需工程師確認後才能決定設計方向」 |
| `[需業務/法務確認]` | 涉及合約責任、退款義務、不可抗力等法律層面，或業務面的政策決策，超出 OP 作業層面 | 風險描述加註「此規則涉及法律責任歸屬或商業政策，需法務或業務主管層級確認，不建議由 PM 自行判斷」 |

> **`[需業務/法務確認]` vs `[雄獅規則－待確認]` 的區分**：
> - 前者是「**誰有資格確認**」的問題 — 答案不在 OP 或工程，需要更高層級的判斷
> - 後者是「**規則是否符合現行實務**」的問題 — 向業務/OP 詢問即可釐清

## 確認後的更新方式

當 PM 告知某條規則已確認，將標注改為：

```
✅ 已確認（YYYY-MM-DD）：[一句話記錄確認內容]
```

例如：
```
✅ 已確認（2026-04-15）：min pax 計算單位為成人人數，不含兒童。
```
